# Civic Compute Framework

**A policy framework for municipal compute rights, sovereign model governance, and accountable AI in public services.**

→ [View the Framework Website](https://dicemanorama.github.io/Civic_Compute_Framework) *(live once GitHub Pages is enabled)*

---

## What This Is

Government at every level is beginning to rely on AI for public services — permitting, social services, transit, public health, and more. Almost all of it runs on commercial APIs owned by private companies, with no public oversight, no data sovereignty, and no protection against pricing changes, model deprecations, or terms-of-service shifts.

This framework proposes a different path: **municipal AI infrastructure as public infrastructure** — negotiated, owned, governed, and improved by the communities it serves.

---

## The Core Idea

### Compute as a Permit Condition
Data centers require enormous power grid allocations, water rights, and local permitting goodwill. Municipalities currently hold rare and time-limited leverage over these facilities. This framework argues that a portion of compute capacity should be negotiated as a public benefit during the permitting process — securing the infrastructure cities will need for public AI services before those services are even built.

### The True Cost Is Inference
Model development is a one-time capital expense. Inference — every query, every decision, every interaction with a public-facing AI system — is a recurring cost that compounds forever. Locking in compute access now, while leverage exists, hedges decades of operational dependency.

### A Three-Layer Governance Architecture

```
[ Federal ]   ← Synthesize outcomes, surface best-performing models, maintain open registry
[ State ]     ← Define standardized output schemas, enable inter-regional sharing
[ Municipal ] ← Develop and maintain models, run inference, incorporate community input
```

States define **what** the output must look like. Municipalities decide **how** to produce it. The federal layer learns from what works across all of them.

### Standardized Outputs, Not Standardized Models
Any model that produces a compliant output schema satisfies the requirement. Local governments retain full autonomy over implementation while data flows upward in a consistent format. The interface is standardized; the implementation is not.

### An Evolutionary Open Commons
Over time, real-world performance data surfaces which models work best for which problems. High-performing models are contributed to a shared open registry. Every jurisdiction benefits from every other jurisdiction's improvements — not through mandates, but through demonstrated merit.

## Democratic Accountability

When the model is the policy, the diff is the platform.

Future elected officials will campaign on specific, auditable proposals: what they would change in the model, what objectives they'd reweight, what outputs they'd require. Unlike today's vague policy promises, these changes are verifiable and trackable. The community — through participatory input processes — sets the objectives and optimization targets before any representative or technologist touches the implementation.

Explainability requirements — what changed and what the downstream effects were — are civic infrastructure, not technical nice-to-haves.

---

## Core Principles

1. **Sovereignty** — Public services must not be architecturally dependent on private interests
2. **Portability** — Standardized outputs, not locked implementations
3. **Legibility** — Citizens can understand, question, and change what the model optimizes for
4. **Evolutionary** — The system learns; better models displace worse ones on merit
5. **Distributed** — No single jurisdiction, vendor, or government layer controls the stack
